# Node SEA / 单文件二进制部署说明

## 1. 说明

本仓库包含 `scripts/build-node-sea.mjs`，用于帮助准备 Node SEA（Single Executable Application）相关配置，但当前实现**只生成模板配置，不自动产出最终可执行文件**。

因此，SEA 在本项目中属于：

- 可选部署优化手段
- 非默认部署方式
- 非性能核心手段

## 2. 当前脚本实际作用

执行：

```bash
npm run sea
```

脚本会：

- 校验 `dist/` 是否存在
- 生成 `sea-config.json`
- 生成用于 SEA 准备的 blob 输出路径配置
- 输出后续手动步骤提示

它不会：

- 自动 patch Node 二进制
- 自动调用 `postject`
- 自动输出最终单文件可执行程序

## 3. 适合使用 SEA 的场景

- 想减少部署时对 Node 运行环境的显式依赖
- 希望交付单文件启动器
- 目标平台和 Node 版本可控

## 4. 不适合夸大 SEA 的地方

SEA 不能替代以下前端性能手段：

- 路由懒加载
- chunk 拆分
- 静态资源长缓存
- index no-cache
- 按需图表加载
- BFF upstream keep-alive

也就是说，SEA 更偏“部署形态优化”，而不是“浏览器首屏性能优化”。

## 5. 推荐态度

对本项目而言，推荐顺序是：

1. 先把默认 Node 生产态跑稳
2. 再引入 Nginx（如有需要）
3. 最后再考虑 SEA 作为交付包装形式

## 6. 风险提示

SEA 是否可行与以下条件强相关：

- Node 版本
- 目标系统架构
- 二进制 patch 工具链
- 第三方依赖的运行方式

因此应将其视为“增强方案”，而不是主路径。

## 7. 结论

当前仓库对 SEA 的支持定位是：

> 为单文件二进制部署提供准备脚本与配置模板，但不默认承担最终打包责任。
