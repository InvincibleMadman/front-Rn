# front-Xe 资产中心重建契约

## 定位

资产中心是现代代码资产浏览器，不是 Dashboard。

它的目标是围绕协议项目 workspace 提供浏览、搜索、索引、预览、下载和关系分析能力，而不是用概览卡片堆叠做仪表盘式展示。

## 视觉参考优先级

1. GitHub Repository Files
2. GitLab File Tree
3. Sourcegraph Code Search
4. Dagster Asset Graph

## 禁止项

- 页面顶部大标题单开成独立仪表盘区块。
- 用 SummaryCard 堆叠承载主要内容。
- 用 Select / Badge 堆叠代替真实浏览和筛选结构。
- 三栏平分布局。
- 自定义一套与主站不一致的主题色体系。
- 暴露服务器真实路径。
- 把 force layout 当作图谱默认方案。

## 布局契约

- 每页最多三栏，且必须存在核心主栏。
- 总览页使用单栏。
- 文件页、搜索页、索引页使用主栏 + 右侧引用栏。
- 关系页以图谱为核心主栏，其余信息附属展开。
- 右侧引用栏展开宽度为 280-320px，收起宽度为 40-48px。
- 右侧引用栏必须稳定挂载，不能因为 open=false 就卸载整棵组件树。
- hooks 不能放在条件分支里，收起状态只隐藏内容，不改变挂载结构。

## 数据展示边界

- 只展示 `workspace://...` 引用和 `virtual_path`。
- 不显示服务器真实文件系统路径。
- 资产、索引、搜索命中和预览都必须以 workspace 语义呈现。

## 性能契约

- 重组件必须 lazy。
- ECharts 只允许出现在总览页和关系页。
- 搜索页、索引页按需加载，不做整页强制图表渲染。
- 内容搜索默认关闭。
- 列表必须分页或“显示更多”，不能一次性铺满所有结果。

## 图谱契约

- 总览页不再使用“单协议成分导图”。总览必须改为整个 workspace / assets catalog 的全局 UML 统计目录。
- 关系页仍保留单协议 detailed lineage，但节点表现必须是 UML 类图实体表，而不是圆角文字节点。
- UML 实体必须包含名称格和属性格，属性行可显示数量、类型、状态、workspace ref 摘要等。
- 优先使用 React + SVG 自绘 UML 画布，不再继续把 ECharts graph 作为默认图谱方案。
- Dagster 风格关系图必须采用固定坐标或类图式布局。
- 禁止把随机散点、自动力导向抖动、CSS 拉伸或大圆角卡片冒充 UML 实体。
- 图谱应突出资产层级和关系，而不是做装饰性动画。

## 画布交互契约

- UML 画布容器必须 `overflow-hidden`，不能依赖浏览器滚动条浏览大图。
- 至少支持整体画布拖动平移。
- 支持滚轮缩放，推荐范围 `0.55 - 1.8`。
- 必须提供“适配视图 / 重置位置”控制。

## 协议切换与导入

- 协议切换框必须支持下拉选择已有协议。
- 导入源码时必须允许输入“新命名协议名”，并将该协议名作为协议根目录名。
- 浏览器侧仍只展示 `workspace://` 语义，不暴露真实服务器路径。

## 冲突覆盖记录

- 旧要求“总览页以当前协议为中心，展示 source/specs/seeds/risk/jobs 节点”已被本次规则覆盖。
- 新总览页改为全局协议目录；单协议详细关系迁移到“关系”页。
- 旧要求“可以使用 ECharts graph”降级为历史兼容参考，不作为总览/关系图默认实现。
